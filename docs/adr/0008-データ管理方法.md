# 0008. データ管理方法

## Status

Accepted

## Context

データは全件取得しても 5MB 程度に収まる見込み。
データ更新頻度は毎日 1 回。
ビルド時にデータを含める方法と API 経由で取得する方法がある。

## Decision

`/public/data/`ディレクトリに Parquet ファイルを配置し、ビルド時に含める方法を選択する

## Consequences

- **メリット**
  - シンプルで実装が容易
  - 初回ロードが高速
  - CDN 経由で配信されるため高速
- **デメリット**
  - データ更新のたびに再ビルド・再デプロイが必要
  - 更新頻度が高い場合、ビルド時間が増える
  - ただし、毎日1回の更新頻度なら問題なし
