# 0003. データ形式の選択

## Status

Accepted

## Context

現時点では、データは全件取得しても数 MB 程度に収まる見込み。
ストレージ効率と読み込み速度はなるべく高めておきたい。
SQL クエリを実行する必要がある。

## Decision

Parquet 形式を選択する

## Consequences

- **メリット**
  - 列指向フォーマットで効率的な圧縮
  - DuckDB-WASM との親和性が高い
  - 高速な読み込みが可能
- **デメリット**
  - R や Python から読み込む場合は、arrow や pandas などのライブラリが必要
    - DuckDB-WASM の場合は組み込み機能で直接読み込めるため、追加のライブラリは不要
