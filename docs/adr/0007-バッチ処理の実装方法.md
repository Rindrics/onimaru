# 0007. バッチ処理の実装方法

## Status

Accepted

## Context

既存の配信サイトからデータを取得するバッチ処理が必要。
データ更新頻度は毎日 1 回。
差分更新の場合、バッチ処理の実行時間は数秒程度の見込み（オーバーヘッド除く）。

## Decision

GitHub Actions を選択する

## Consequences

- **メリット**
  - 無料で利用可能
  - 柔軟なスケジューリング設定が可能
  - 実行時間制限が緩い（数秒の処理なら問題なし）
  - データ更新時に GitHub にコミットし、Vercel が自動デプロイ（[ref](./0002-ホスティングプラットフォームの選択.md)）
- **デメリット**
  - GitHub Actions の知識が必要
